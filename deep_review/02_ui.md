# Review Checklist: UI

## Purpose

Line-level review for UI correctness, accessibility, reusability, layout stability, and render performance.

## Checklist

* Each widget: single responsibility — complex build methods should be refactored.
* Avoid heavy work inside `build()` (no calculations, no I/O).
* Prefer `const` where possible for immutable widgets.
* Check for excessive nesting — consider composition or helper widgets.
* Verify `RepaintBoundary` usage for expensive subtrees.
* Verify scrolling performance (no large images or synchronous work in scroll).
* Accessibility: semantics, labels, tap target sizes, color contrast.
* Text scaling: ensure `MediaQuery.textScaleFactor` respected.
* Avoid `setState` on large parents — scope state lower.
* Use `Hero` tags safely (unique).
* Images: use `cacheWidth`/`cacheHeight` or resized assets for large images.

## Common UI anti-patterns

* `FutureBuilder` or network call in top-level `build()`.
* Using `Opacity` with large subtrees instead of `AnimatedOpacity`.
* Rebuilding entire pages due to small state change.

## Automated checks to run

* `flutter analyze` (Run: Completed)
* `flutter test` (Run: Pending fixes for test suite)
